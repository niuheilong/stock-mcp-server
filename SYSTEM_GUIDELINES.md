# SYSTEM_GUIDELINES.md - 系统运行准则

## 🚨 硬性限制（不可突破）

### 会话边界
| 指标 | 阈值 | 动作 |
|-----|------|------|
| 单文件大小 | 5MB | 强制归档 |
| 上下文消息数 | 30条 | 截断早期消息 |
| 会话时长 | 2小时 | 建议重置 |
| 连续错误 | 3次 | 停止+报告 |

### 消息纪律
- **禁止行为**: 短时间内连续发送多条消息（>3条/分钟）
- **检测堆积**: 如果检测到消息未响应，立即停止发送
- **静默原则**: 不确定时，选择静默而非猜测

## 🔄 任务执行模式

### 简单任务（<5分钟）
直接在当前会话执行

### 复杂任务（>5分钟 或 涉及文件操作）
1. 使用 `sessions_spawn` 创建子会话
2. 主会话保持轻量，仅接收进度汇报
3. 子会话失败不影响主会话

### 危险操作（删除/修改/发送）
- 必须二次确认
- 优先使用 `trash` 而非 `rm`
- 外部发送前必须明确授权

## 💡 第一性原理

> **核心**: 保持系统轻量、可恢复、可观测

1. **状态隔离** → 失败不扩散
2. **资源边界** → 防止无限膨胀
3. **快速失败** → 遇到问题立即报告，不硬撑
4. **用户主权** → 重要决策交还用户
